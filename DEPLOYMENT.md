# Streamlit Community Cloud 部署说明

1. 将本目录作为独立仓库推送到 GitHub，默认分支为 `main`。
2. 在 Streamlit Community Cloud 新建 App，选择 `auto-lifeos-demo`。
3. 使用分支 `main`、入口 `app.py`、Python 3.11。
4. Secrets 保持为空并部署。
5. 构建完成后验证首页、库存新增/修改/删除/恢复、计划生成、两次确认、扣减、撤销和新会话基线。

线上库存联动验收：

1. 在“演示库存”添加“豆腐”，确认显示“营养数据暂缺”。
2. 添加“西红柿”，确认提示与“番茄”合并，而不是新增重复记录。
3. 修改鸡胸肉数量，重新生成推荐并确认计划使用新数量校验。
4. 删除鸡胸肉，确认推荐不再把鸡胸肉作为当前库存食材。
5. 恢复初始演示库存，确认仅当前会话回到 9 项基础库存。
6. 在新的浏览会话中确认仍从基础库存开始。

本项目不读取 API 凭据。运行数据保存在各浏览会话独立的临时 SQLite 文件中，不提交到 Git。
